WINTER 2023 COURSE PROJECT/PERSONAL PROJECT

Mida's Portfolio


Fully developed website

SUMMARY


To showcase my abilities learned within my SI 339 (Web Design, Development, and Accessibility) course—I fully developed this website with the usage of HTML, CSS and javascript.

The project simply asked for 15-20 hours worth of work; however, I went above and beyond to incorporate both UX/UI design and front-end development for a complete website portfolio.

ROLE


UX Engineer


TEAM


Myself!


TOOLS


Adobe Illustrator, Canva, Figma, VSCode


DURATION


Apr 2024


WHAT WAS THE OBJECTIVE?

To complete a coding project that demonstrated accesible front-end programming. My goals for this project were to: Create a brand identity for myself, code using HTML, CSS, and JavaScript and utilize responsive design within my final deliverable.




RESEARCH PHASE


To create an effective portfolio that not only displays my professional side but also my personal interests and aesthetics, I scavenged to find inspiration. From my findings, I was able to pinpoint key elements of good portfolios:



branding

effective portfolios each had something in common—their own personal brand that stayed consistent throughout their sites

highlights

in order to attract attention and sell yourself to viewers, i noticed that important info should be displayed boldly for skimming

storytelling + detail

providing each step of the design process is important to providing not only the decision itself but also the psychological reasoning behind those inclusions

DESIGN PHASE


Simplistic but recognizable brand identity — red primary color for logo with a diamond shaped flower standalone

Needed a solid mockup on Figma before coding the front-end for better organization and setup

Because of limited time, needed to prioritize some aspects more than others and to make design decisions that are easily customizable for the future.

DESIGN GUIDELINES + LOGO IDEATION


CHALLENGES

☹

Fully developing the website from scratch—HTML, CSS and Javascript. The structuring of the pages and keeping up with the different styling pages and different divs was difficult.

☹

Making sure the website was both desirable and accesible was also challenging to keep track of and have all the details in track for responsive design.

☹

Having a final deliverable completed in the time frame given (about 3 weeks) was also difficult. Although it was supposed to be 15-20 hours, the project definitely took more time due to the complexity.


SUCCESSES

☺

Replicating the Figma prototype pretty similarly despite developing from scratch for the first time and also overcoming challenges such as color contrast when building the website.

☺

Learning from mistakes within the coding portion and applying the lessons learned from seeking outside resources to debug was another success.

☺

Incoporating more than expected from the figma prototype and incorporated javascript to make the website more dynamic than static.

KEY TAKEAWAY



😅


Organization is very essential to a project. I think a bad foundation to any type of project will set one up for failure. This is why both components of UX and development are pivotal for a successful website project.
